# 基礎知識複習

本節複習微處理機課程所需的基礎知識，包括數字系統、微控制器歷史、ARM 架構等。

## 十六進制系統 (Hexadecimal System)

十六進制是以 16 為基數的計數系統，使用 0-9 和 A-F 共 16 個符號表示數值。在電腦科學中，十六進制常用作二進制數的便捷表示法。

### 十六進制與二進制轉換

- **二進制轉十六進制**：從右至左，每 4 位二進制數字轉換為 1 位十六進制數字
  - 例：`1101 0111` (二進制) = `D7` (十六進制)
  
- **十六進制轉二進制**：每位十六進制數字轉換為 4 位二進制數字
  - 例：`3A` (十六進制) = `0011 1010` (二進制)

### 在程式中表示十六進制

- C 語言：使用 `0x` 前綴，如 `0x1A`
- ARM 組合語言：使用 `0x` 前綴，如 `MOV R0, #0x1A`
## 微控制器歷史

### 微處理器 vs 微控制器

- **微處理器 (Microprocessor)**：
  - 僅包含 CPU 核心
  - 需外接 RAM、ROM、I/O 等
  - 例如：早期的 Intel 8086、Motorola 68000

- **微控制器 (Microcontroller, MCU)**：
  - 在單一晶片上整合 CPU、RAM、ROM、I/O 週邊等
  - 適合嵌入式系統應用
  - 例如：8051、PIC、AVR、STM32
### 選擇微控制器的考量因素

1. **晶片特性**：
   - 時脈速度
   - 功耗
   - 價格
   - 內建記憶體大小
   - 週邊種類和數量

2. **可用資源**：
   - 開發環境 (IDE)
   - 編譯器
   - 軟體庫
   - 技術支援
   - 供應商可靠性

## ARM 歷史與架構

### ARM 的起源

- 源自英國 Acorn Computers 公司
- 早期稱為 Acorn RISC Machine
- 後改名為 Advanced RISC Machine

### ARM 商業模式

- ARM 採用 IP 授權模式 (矽智財)
- ARM 設計處理器核心，授權給其他公司生產晶片
- 主要客戶包括：ST、TI、NXP、Apple 等

### ARM 應用領域

- 嵌入式系統
- 行動裝置 (手機、平板)
- 物聯網裝置
- 汽車電子
- 消費性電子產品

### ARM Cortex 系列

Cortex 系列是針對微控制器市場的 32 位元核心：

- **Cortex-M**：微控制器應用 (M0/M0+/M3/M4/M7)
- **Cortex-A**：應用處理器 (手機、平板等)
- **Cortex-R**：即時系統

### ARM Family vs ARM Architecture

- **ARM Family (系列)**：特定產品線，如 ARM7、ARM9、ARM11
- **ARM Architecture (架構)**：指令集架構版本，如 ARMv6、ARMv7、ARMv8
- 兩者不可互換，例如 ARM11 Family 基於 ARMv6 架構

## STM32 家族

### STM32 簡介

STM32 是 STMicroelectronics (意法半導體) 基於 ARM Cortex-M 核心的 32 位元微控制器產品線。

### STM32 命名規則

STM32 + 系列 + 子系列 + CPU核心 + 引腳數 + Flash大小 + 封裝

例如：STM32**F**1**03**C**8**T6
- **F**：系列 (F = 通用型)
- **1**：子系列 (性能等級)
- **03**：特性碼
- **C**：引腳數 (48 針)
- **8**：Flash 大小 (64KB)
- **T6**：封裝類型 (LQFP48)

### STM32 系列特性

- **F0**：入門級，基於 Cortex-M0
- **F1**：主流型，基於 Cortex-M3
- **F2/F4**：高性能型，基於 Cortex-M3/M4
- **F7**：高端型，基於 Cortex-M7
- **L0/L1/L4**：超低功耗型
- **H7**：超高性能型，基於 Cortex-M7

### STM32 開發板範例

- **NUCLEO**：官方開發板，價格適中，多種型號
- **DISCOVERY**：功能豐富的評估板
- **Blue Pill**：非官方廉價開發板，基於 STM32F103C8T6
- **OpenCR**：機器人控制板
- **Pixhawk**：無人機控制板
